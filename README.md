# PM Wiki LLM

База знаний по курсу «Управление проектами в IT», построенная по трёхслойной архитектуре: `raw/` (исходники) → `wiki/` (vault для Obsidian) → `CLAUDE.md` (схема и правила для агента). Подробности об архитектуре и слоях — в [CLAUDE.md](CLAUDE.md).

## Как открыть в Obsidian

### Шаг 1. Установить Obsidian
Скачать: [obsidian.md](https://obsidian.md) → установить под свою ОС (macOS / Windows / Linux).

### Шаг 2. Получить vault
Vault находится в этом репозитории — внутри папки `wiki/`.

**Вариант A — через Git (рекомендуется):**
```
git clone https://github.com/siarhei-nikanenka/PM-wiki-LLM
```
После клонирования vault будет по пути: `PM-wiki-LLM/wiki/`

**Вариант B — скачать архив:**
GitHub → репозиторий `PM-wiki-LLM` → кнопка `Code` → `Download ZIP` → распаковать → папка `wiki/`

### Шаг 3. Открыть vault
1. Запустить Obsidian
2. Нажать **Open folder as vault**
3. Выбрать папку `wiki/` (не корень репозитория)
4. Подтвердить «Trust author and enable plugins»

### Шаг 4. Что внутри
Vault (`wiki/`) состоит из трёх разделов:

- **`sources/`** — конспекты лекций курса (по одной странице на каждое занятие: PM 1 Intro, PM 2 Main Notions, PM 3 Methodologies, PM 4 Agile Methodologies, PM 5 Scrum). Каждая страница: резюме занятия, ключевые тезисы, практические задания, ссылка на исходник в `raw/`. Хорошая точка входа, если хочешь зафиксировать главное по конкретной лекции.
- **`concepts/`** — тематические страницы: концепты, методологии, практики (Agile, Scrum, Kanban, XP, Lean, Iron Triangle, типы контрактов и т.д.). Это «энциклопедия»: каждая страница объясняет один концепт — его суть, связь с другими темами, ссылку на источник. Хороший способ быстро разобраться с незнакомым термином.
- **`entities/`** — карточки людей, организаций и продуктов курса (преподаватель, сам курс, PMBOK). Полезно, когда встречаешь упоминание и хочешь понять контекст.

Дополнительно:
- **`index.md`** — каталог всех страниц вики с кратким описанием содержания каждой.
- **`overview.md`** — общий синтез накопленных знаний по теме.
- **`log.md`** — журнал операций над вики (ingest/query/lint).

### С чего начать
Используй `Cmd/Ctrl + O` → введи название концепта (например, `scrum` или `agile`) → Obsidian найдёт страницу. Оттуда переходи по ссылкам — граф раскрывается сам. Либо начни с [`index.md`](wiki/index.md) или [`overview.md`](wiki/overview.md).

### Советы по навигации
- Быстрый поиск по всему vault: `Cmd/Ctrl + O`
- Граф связей между заметками: `Cmd/Ctrl + G`
- Поиск по тексту: `Cmd/Ctrl + Shift + F`

## Как установить и запустить Claude Sidebar

Community-плагин **Claude Sidebar** (автор Derek Larson) позволяет работать с Claude Code прямо в боковой панели Obsidian, не выходя из vault. Доступен только на десктопе (macOS / Windows / Linux), для мобильной версии Obsidian не работает.

### Предварительное условие
На компьютере должен быть установлен и авторизован [Claude Code](https://docs.claude.com/en/docs/claude-code) (CLI) — плагин запускает его как процесс под капотом. Проверить, что CLI работает:
```
claude --version
```

### Шаг 1. Включить community-плагины
1. Открыть **Settings** (⚙️) → **Community plugins**
2. Если включён Restricted mode — нажать **Turn on community plugins**

### Шаг 2. Установить Claude Sidebar
1. В **Community plugins** нажать **Browse**
2. В поиске ввести `Claude Sidebar`
3. Открыть страницу плагина → **Install**
4. После установки нажать **Enable**

### Шаг 3. Настроить плагин
1. Перейти в **Settings** → **Community plugins** → **Claude Sidebar** (иконка шестерёнки рядом с плагином)
2. Указать путь к исполняемому файлу `claude`, если плагин не нашёл его автоматически (узнать путь можно командой `which claude` в терминале)
3. При первом запуске плагин может попросить авторизоваться — пройти стандартный логин Claude Code (тот же, что используется в терминале)

### Шаг 4. Открыть и использовать
1. Открыть боковую панель: иконка Claude в левой/правой боковой панели Obsidian, либо команда **Open Claude Sidebar** через `Cmd/Ctrl + P`
2. В панели появится чат с Claude Code — он видит открытый vault (`wiki/`) как рабочую директорию, поэтому может читать и редактировать страницы вики прямо по запросу, придерживаясь правил из [CLAUDE.md](CLAUDE.md)
3. Для ingest нового источника, query или lint (см. workflow в [CLAUDE.md](CLAUDE.md)) достаточно сформулировать задачу в чате Sidebar — так же, как в обычной сессии Claude Code в терминале
