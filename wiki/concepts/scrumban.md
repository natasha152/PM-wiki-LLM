# Scrumban

Из [[pm-4-agile-methodologies]].

Гибридная методология, объединяющая стратегическую структуру [[scrum]] (спринты, планирование, ретроспективы, чёткие роли) с тактической гибкостью [[kanban]] (доска с WIP-лимитами для управления задачами внутри спринта).

Команда работает циклами-спринтами как в Scrum (планирование целей, регулярный анализ результатов), но внутри спринта задачи перемещаются по Kanban-доске (например, Planned → Dev → Test → Done) с ограничениями по колонкам, чтобы не допускать узких мест. Используются Daily Scrum, Increment, Sprint Review.

## Когда выбирать Scrumban
Команда работает по Scrum, но сталкивается с частыми изменениями задач в середине спринта (например, срочные баги от пользователей, которые невозможно игнорировать); нужен гибкий инструмент управления задачами внутри спринта, при этом важно сохранить стратегическую направленность Scrum.
