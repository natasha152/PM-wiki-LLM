# Оценка проекта

Из [[pm-2-main-notions]].

Длительность проекта ≠ суммарная трудоёмкость: параллельное выполнение задач сокращает сроки, но добавление людей сверх меры увеличивает накладные расходы на коммуникацию.

Оценка ведётся по четырём параметрам:
1. **Ресурсы** — люди, оборудование, лицензии.
2. **Время** — estimated → planned → actual.
3. **Экспертиза команды** — Senior vs Junior влияет на скорость в 2–3 раза.
4. **Деньги** — рейты × часы + резерв на риски.

В Agile-проектах стоимость считается через итерации — см. [[velocity-metrics]] (стоимость спринта = количество человек × рабочие часы × ставка).
