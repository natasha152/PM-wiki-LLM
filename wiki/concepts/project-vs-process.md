# Проект vs процесс

Из [[pm-2-main-notions]].

По [[pmbok]], **проект** — временное предприятие, направленное на создание уникального продукта, услуги или результата. Два ключевых признака:
1. **Временной характер** — чёткие дата начала и окончания.
2. **Уникальность результата** — даже при копировании продукта результат будет иным (другая команда, другой ландшафт, накопленный опыт).

Проект завершается при достижении целей, исчерпании ресурсов/финансирования, отпадении потребности или по юридическим причинам.

**Процесс** — повторяемый набор действий для поддержания стабильного, стандартизированного результата (например, ежемесячная выплата зарплаты).

Улучшение процесса само по себе может быть проектом: как только новый процесс внедрён — проект закрывается, а результат становится новым процессом.
