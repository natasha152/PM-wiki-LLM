# CI/CD — Continuous Integration / Continuous Delivery / Deployment

Из [[pm-6-sdlc]].

Три связанные практики автоматизации, которые ускоряют и делают безопасной доставку кода от разработчика до конечного пользователя.

---

## Continuous Integration (CI)

Автоматическая **сборка и тестирование кода** при каждом изменении (commit/push). Позволяет команде обнаруживать конфликты и баги немедленно, а не на этапе финального merge.

Инструменты: Jenkins, GitLab CI, GitHub Actions.

---

## Continuous Delivery

Автоматическая **подготовка к развёртыванию в production** после прохождения CI-этапа. Финальный шаг — деплой в production — выполняется **вручную** (ручное подтверждение).

Когда нужно: регуляторные требования, необходимость ручного контроля релиза (например, финтех со стороны юридического отдела).

---

## Continuous Deployment

**Полностью автоматическое** развёртывание в production без ручного утверждения. Каждый успешно прошедший тесты commit автоматически попадает к пользователям.

Когда нужно: высокая зрелость команды, сильное покрытие тестами, нет регуляторных ограничений.

---

## Сравнение

| | CI | Continuous Delivery | Continuous Deployment |
|---|----|--------------------|----------------------|
| Автосборка + тесты | ✓ | ✓ | ✓ |
| Авто-подготовка к prod | — | ✓ | ✓ |
| Ручное подтверждение деплоя | — | **Да** | Нет |
| Авто-деплой в prod | — | — | ✓ |
