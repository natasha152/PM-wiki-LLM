# PM Wiki LLM — Schema

Эта база знаний построена по трёхслойной архитектуре (raw → wiki → schema). Этот файл — schema-слой: правила, структура и рабочие процессы, которым должен следовать агент.

## Слои

- `raw/` — неизменяемые исходники (статьи, PDF, изображения). Источник истины. Никогда не редактировать и не удалять файлы здесь без явной просьбы пользователя.
  - `raw/articles/` — статьи, заметки, веб-страницы
  - `raw/papers/` — статьи/доклады в формате PDF
  - `raw/assets/` — изображения и прочие медиа, на которые ссылаются источники
- `wiki/` — полностью управляется агентом. Здесь живут резюме, страницы сущностей, концептуальные страницы и синтез.
  - `wiki/index.md` — каталог всех страниц вики с однострочными описаниями
  - `wiki/log.md` — апенд-онли журнал операций
  - `wiki/overview.md` — общий синтез накопленных знаний
  - `wiki/sources/` — по одной странице-резюме на каждый источник из `raw/`
  - `wiki/entities/` — страницы про людей, организации, продукты, инструменты и т.п.
  - `wiki/concepts/` — страницы про идеи, методы, паттерны, сравнения
- `CLAUDE.md` (этот файл) — schema-слой, конфигурация агента

## Роли

Человек курирует: выбирает, какие источники добавлять, и задаёт вопросы.
Агент обслуживает: суммирует, расставляет кросс-ссылки, ведёт журнал, поддерживает индекс в актуальном состоянии.

## Workflow: Ingest

Когда в `raw/` добавлен новый источник:

1. Прочитать источник целиком.
2. Кратко обсудить с пользователем ключевые моменты (если не указано иное — можно сразу резюмировать).
3. Создать страницу в `wiki/sources/<slug>.md` с резюме и ссылкой на оригинал в `raw/`.
4. Обновить `wiki/index.md` — добавить запись в нужную секцию.
5. Пересмотреть связанные страницы в `entities/` и `concepts/`: создать новые или обновить существующие, если источник добавляет к ним информацию.
6. При необходимости обновить `wiki/overview.md`, если источник существенно меняет общую картину.
7. Добавить запись в `wiki/log.md`:
   ```
   ## YYYY-MM-DD ingest | <название источника>
   создана wiki/sources/<slug>.md, обновлены: ...
   ```

## Workflow: Query

Когда пользователь задаёт вопрос:

1. Найти релевантные страницы через `wiki/index.md` (и при необходимости grep по `wiki/`).
2. Синтезировать ответ, опираясь на найденные страницы, с указанием, на какие страницы/источники ссылается ответ.
3. Если ответ содержит новый устойчивый синтез, которого ещё нет в вики — предложить пользователю создать новую страницу (`concepts/` или обновление `overview.md`).
4. Если была создана новая страница — обновить `index.md` и добавить запись в `log.md` с операцией `query`.

## Workflow: Lint

Периодически (по просьбе пользователя или при заметных изменениях):

1. Искать противоречия между страницами.
2. Искать устаревшие данные (страницы, которые противоречат более новым источникам).
3. Искать страницы-сироты — без входящих ссылок из `index.md` или других страниц.
4. Искать недостающие кросс-ссылки между явно связанными страницами.
5. Зафиксировать находки и исправления в `wiki/log.md` с операцией `lint`.

## Соглашения

- Все файлы вики — markdown, имена файлов — `kebab-case` slug.
- Каждая страница в `wiki/` должна явно ссылаться на исходники в `raw/`, на основе которых она написана.
- `index.md` — content-first: не просто список файлов, а краткое описание содержания каждой страницы.
- `log.md` — только дописывается (append-only), не редактируется задним числом.
- `overview.md` всегда должен отражать совокупность прочитанного, а не только последний источник.
